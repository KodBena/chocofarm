THE PROBLEM: adaptive stochastic orienteering / belief-MDP. 20 treasures, exactly 5 present per run (uniform w/o replacement → 15,504 latent "worlds"; i.i.d. re-roll). 16 OVERLAPPING detection regions → binary DISJUNCTIVE observations; 4 δ treasures (observe==collect). Objective = long-run rate = treasures/time (renewal-reward) via Dinkelbach (λ-penalized value). Belief = the surviving-world set (observable; the policy acts on it). KEY FINDINGS: a clairvoyant policy (free perfect info) scores +70% over static — large value-of-information — but every model-based search tried (greedy, certainty-equivalent, rollout, sparse-sampling, NMCS L1/L2, ISMCTS) sits BELOW the static floor, capturing ~none of the +70%, because (a) the VoI is gated behind deep contingent sensing chains under weak disjunctive detectors, and (b) determinized search is optimistically biased (NMCS got WORSE with depth — winner's curse). The AlphaZero hypothesis: a LEARNED value/policy over the belief would (i) amortize search to go deeper cheaply, and (ii) replace optimistic determinized rollouts with a calibrated value — attacking both failure modes.

INVESTIGATE the design space, covering at least:
- **Belief featurization (the crux):** the net must read the belief. Per-treasure marginal posteriors (20 numbers) are cheap but LOSSY — they drop the joint/clause structure disjunctive detections create (exactly what lets you deduce a location from two ambiguous reads). Consider richer encodings (pairwise/cluster stats; the accumulated observation-clause set; a set-encoder/DeepSets over surviving worlds or over the detector-outcome history; |belief|), plus location and collected-mask. What is the minimal SUFFICIENT statistic vs a lossy-but-practical one?
- **Architecture** (MLP over features; set/transformer encoder; size).
- **Output heads:** policy over the variable/masked legal action set (20 treasures + 16 detectors + terminate); value head — and crucially HOW to define the value target under the RATE objective (learn the λ-penalized value at fixed λ? a λ-conditioned value? learn the rate directly?). Address this Dinkelbach-vs-value-target question concretely.
- **The guided search:** single-agent, partially observed, STOCHASTIC (disjunctive outcomes = chance nodes). ISMCTS with net priors+value (the running ISMCTS is the natural expert) vs Gumbel-AlphaZero low-simulation MCTS. Chance-node handling.
- **Expert-iteration loop** (no self-play — single agent): generate trajectories via net-guided search on the simulator, train on search-improved targets, iterate. Data, targets, loss, schedule.
- **Evaluation** against `realizable_static` (floor) and `clairvoyant_rate` (ceiling) — % of the +70% clawed back.
- **Feasibility/cost** on a single 4-core CPU machine (argue if a GPU is warranted): is it tractable, and what is the SMALLEST experiment that decides whether a learned value beats the shallow pack?
