THE ERROR (facts; investigate independently):
- `env.py` builds detector cover sets as `cover_mask[i] = {i} ∪ {j : region Δ_i overlaps region Δ_j anywhere}` — using the `overlaps` array (17 pairwise area-overlaps) from `chocobo_instance.json`. It treats "entering region i" as revealing the DISJUNCTION over treasure i AND ALL regions Δ_i overlaps, simultaneously. (env.py's own comment flags this as "an over-approximation; per-arrangement-face reification is a later refinement.")
- The static-optimization design doc (`docs/design/static-shortcuts.md`, and `docs/agents/static-opt-report.md`) used this to assert a detector covering the NW cluster **{8,9,10,11,12}** and a headline "**5.16× early-negative collapse** (D_8/D_9 negative → 15,504→3,003 worlds, rules out all of {8,9,10,11,12})", plus a two-level cluster decomposition built on these cover sets.

The largest-cardinality intersection is **Q = Δ_8 ∩ Δ_9 ∩ Δ_11 ∩ Δ_12**, which is a **tiny sliver**, and **Δ_10 is not even close to Q**. So the true disjunctive read depends on the ARRANGEMENT FACE you stand in (which subset of polygons covers that exact point), NOT the union of all of a region's pairwise overlaps.
- Prior context: consult-001 caught the OPPOSITE error earlier — a single-`representative_point()` cover that UNDER-counted overlaps (singletons). The current `cover_mask` was the "fix" for that, and it over-shot to this over-approximation.
