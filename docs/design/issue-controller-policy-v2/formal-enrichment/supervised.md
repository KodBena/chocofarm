# Supervised learning — formal supplement & prior art

> Family scope: supervised

## C0 — Constant / threshold baselines (all-allow + hand-fixed single-feature threshold)

**Formal object.** Under the model, all-allow is the EXACT identity of the conforming closed two-party protocol of §2: the DENY-only gate never fires, so every RELY/GUARANTEE pair discharges as the model derived them and the plant runs at its native operating point — the §5-R1 negative-feedback batch-size fixed point min(T·K, max_batch+K−1) (§6), with the §3.3 self-batching loop (DOF-F) free-running. Formally all-allow is the model's 'no-control' reference dynamics: state = the §3 nondeterministic-timing state (per-slot δ_park streams superposed into the eligible-set, §3.1; service S in its shape-band, §3.2), action ≡ 1 (null), objective = the realized throughput of that uncontrolled fixed point. The hand-fixed threshold is a stationary deterministic policy π(x[t]) = ¬(ready_backlog_norm[t] ≥ θ ∧ inflight[t] < D): a fixed partition of the per-thread projection of the §3.1 eligible-set state into {allow, deny}, whose deny half intentionally pushes DOF-B (coalescing degree) up by holding ready slots back so the next issue gathers more rows.

**Reduction / algorithm fit.** all-allow is not an algorithm to reduce but the model's reference execution — it is the ONLY policy for which the §2 RELY R4 (liveness) and the §6 boundedness proof transfer verbatim, because it adds no new deny-induced timing. That makes it the formal control arm in the precise sense ADR-0009 wants: any learned-policy dps claim is a claim that the controlled fixed point beats the §6 uncontrolled fixed point. The threshold is the degenerate (zero-capacity) member of every learner below — formally the 1-rule decision stump, i.e. the depth-1 special case of the C2 tree and the hard-threshold limit of the C1 logistic — so it is the right thing to subtract from each. No transformation is needed; the reduction is 'identity = the model's native dynamics'.

**DOF moved (depth>1 live; approximate per caveat).** all-allow moves NOTHING (it is the model's reference). The threshold's deny half moves DOF-B (coalescing degree, approximate: holds ready slots so |eligible-set| at the next issue is larger) and, only because chunk_floor un-pins the §0 spine, DOF-depth/D (approximate: a held thread can now stand at inflight∈[1,D] rather than the model's identically-1, so the inflight<D term in the threshold becomes live rather than the dead knob §0/DOF-depth made it). Flag: the model's depth==1 makes the threshold's inflight<D conjunct vacuous; under the caveat (chunk_floor on) it is live, so the threshold acquires real grip the model could not see.

**What the model adds vs v2.** The model upgrades 'all-allow is the A/B control' from a fixture fact (issue_engine identity) to a PROVEN equivalence: §2 + §6 show all-allow reproduces the derived fixed-point dynamics exactly, so the dps delta is the honest figure of merit for a structural reason, not just a byte-for-byte one. It also corrects the v2 claim that the threshold 'often captures most of the gain': §6's negative feedback means the uncontrolled fixed point already self-clocks toward min(T·K, max_batch+K−1), so a static threshold can only help where the system would otherwise sit in the §5-R4-like coalescing-collapse basin (the README empirical convoy: B:88→1, rows/forward ~130→1-3) — i.e. the threshold's win is basin-avoidance, not steady-state batch-fattening. That is the single most important correction to the whole family's framing.

## C1 — Per-thread logistic / linear gate (shared weights, PRIMARY recommendation)

**Formal object.** The gate decision becomes a STATIONARY MEMORYLESS POLICY π_w: R^F → {0,1} over the per-thread projection of the model's §3 timing state. State per thread = the observable coordinates of §3.1's eligible-set / §3.2's coalescing statistics (ready ≈ |eligible-set| at the snapshot instant; coalesce_degree_inst ≈ the realized DOF-B; inflight ≈ the now-live DOF-depth under the caveat; leaf_rate ≈ a projection of throughput at the §6 fixed point). Action = allow/deny, the DENY-only lever of §2 that perturbs the §3.3 closed loop. Dynamics = the §6 self-clocking negative feedback (so a STATIONARY policy is well-posed: the plant HAS a fixed point to sit at — the model grants exactly the memorylessness C1 assumes). Objective = realized dps, which under §6 is a monotone-increasing function of where the batch-size fixed point min(T·K, max_batch+K−1) sits AND of staying out of the §5-R4 collapse basin. So C1 is fitting the indicator of the high-throughput basin's stationary gate.

**Reduction / algorithm fit.** Karp-style: the supervised fit reduces to LINEAR SEPARATION of the model's state space into {gate-on-good, gate-off-good}, with the §6 fixed-point structure supplying well-posedness (a stationary optimal gate EXISTS because the plant is a contraction toward a fixed point, §6 confidence-high on boundedness). The transformation that makes ordinary logistic regression apply: (i) per-thread parameter-sharing is justified by the model's PERMUTATION SYMMETRY across DEALERs — §2.2/§2.3 give every DEALER an identical RELY/GUARANTEE contract and §3.1 makes the K=N·base slot streams exchangeable, so one shared w over T exchangeable rows is the correct invariant model class, not merely a convenience. (ii) first-difference of the cumulative msgs/leaves (the file's preprocessing) is the model-mandated map from §0's monotone counters to the rate coordinates §6 actually governs. The reduction is exact for the LINEAR part; it FAILS to capture the §5-R4 bistability (a logistic boundary is convex/connected, but the collapse basin is a measure-small non-convex tail per the README 'metastable tail'), which is precisely C1's stated 'misses interactions' con, now given a formal cause.

**DOF moved (depth>1 live; approximate per caveat).** C1's deny moves DOF-B (coalescing degree: holds ready, fattens the next §3.2 forward — approximate, the intended lever) and DOF-depth/D (approximate, live ONLY under the caveat/chunk_floor: a held thread occupies inflight∈[1,D], reactivating the §0-dead D and, per §2.6.4/DOF-R, a non-vacuous intra-thread reorder that depth==1 forbade). Indirectly it moves DOF-F (self-batching feedback): by biasing per-thread emission timing it shifts which messages coalesce during a server forward (§3.3 'no recv during a forward'), nudging the §6 fixed point. It does NOT move DOF-S (service-time band) directly — S is shape-determined (§3.2), and C1 only changes the shape via DOF-B. Flag all as approximate: the model pinned DOF-depth at 1; these readings hold only in the superseded depth>1 regime the caveat names as live.

**What the model adds vs v2.** Three things the unscaffolded v2 could not state. (1) It JUSTIFIES the memoryless-linear choice formally: §6's negative-feedback fixed point means the plant is (modally) a stationary contraction, so a state→action map with no temporal memory is the right class for the R1 modal regime — the v2 picked this by analogy; the model derives its well-posedness. (2) It JUSTIFIES parameter-sharing as the model's DEALER permutation symmetry (§2.2/§2.3 + §3.1 exchangeable slot streams), upgrading 'generalizes across T' from convenience to invariance. (3) It NAMES C1's ceiling precisely: the cons say 'linear misses interactions'; the model says the missed structure is the §5-R4 / convoy collapse basin (a non-convex metastable tail, README) — a linear boundary provably cannot separate a measure-small non-convex basin, so C1 will recover the R1-vs-R3 amortization signal but systematically miss the rare collapse the README identifies as the real failure. That redirects C1's value: it is a good R1-operating-point gate, NOT a collapse-catcher.

## C2 — Gradient-boosted / random-forest tree ensemble (per-thread row classifier)

**Formal object.** Same stationary memoryless per-thread policy π: R^F → {0,1} over the §3 state projection as C1, but the hypothesis class is axis-aligned-partition (decision regions = unions of boxes in the model's state coordinates). Under the model this is the natural class for the gate because the plant's regime structure §5 is itself a PARTITION of the operating space: R1 (steady coalescing), R2 (saturation, T·K>max_batch), R3 (starved/arrival-bound), and the §5-R4-like collapse basin are separated by THRESHOLDS on exactly the observable coordinates (max_batch boundary on B; T·K vs max_batch; ready-backlog level vs the eligible-set size). Action/dynamics/objective identical to C1: DENY-only perturbation of the §3.3 loop, §6 self-clocking dynamics, dps objective tracking the fixed-point location and basin identity.

**Reduction / algorithm fit.** Karp-style: fitting a tree ensemble reduces to learning the §5 REGIME PARTITION and assigning a gate per cell. The transformation that makes GBDT/RF the right tool: the model's regimes are defined by CONJUNCTIONS of threshold tests on the observables — R2-OVERSHOOT is 'T·K > max_batch ∧ last-message crosses the cap' (§5-R2, two thresholds); the collapse basin is 'high ready_backlog ∧ low coalesce_degree_inst ∧ low leaf_rate' (the README convoy fingerprint, a 3-way conjunction). Axis-aligned trees represent conjunctions of thresholds natively and exactly, whereas a linear C1 boundary cannot. So the reduction is: model-regime-membership ≅ a decision-tree path, hence GBDT/RF is the hypothesis class isomorphic to §5's partition geometry. This is a STRICT formal improvement over C1 for the collapse basin: a shallow tree (depth 3-6, the file's pick) can carve the non-convex measure-small §5-R4 tail that the convex logistic provably cannot. The reduction's limit: trees are still memoryless, so they classify the basin's CURRENT footprint but not its metastable TRAJECTORY (the README 'fails to recover and sticks' is a temporal property no static partition captures — that needs C-NEW-1 or C3).

**DOF moved (depth>1 live; approximate per caveat).** Identical to C1 in WHICH DOFs move (DOF-B coalescing; DOF-depth/D live under the caveat; DOF-F via emission-timing shift; not DOF-S directly), because the action space is unchanged — C2 differs only in the policy's functional form, not its actuation. The model adds that C2's partition can be ALIGNED to the DOF-defined regime boundaries: it can place a split exactly at the §5-R2 overshoot threshold (T·K vs max_batch) and at the collapse-basin coalesce_degree floor, so its DOF-B/DOF-F actuation can be made regime-CONDITIONAL (fatten aggressively in R3-arrival-bound, back off near the R2 cap) in a way C1's single linear pressure cannot. Flag DOF-depth/D as the approximate caveat reading (model pins it at 1; live only under chunk_floor).

**What the model adds vs v2.** The model converts C2's vague 'captures interactions' advantage into a SPECIFIC structural claim: the interactions that matter are the §5 regime-boundary conjunctions (max_batch overshoot threshold; the convoy-collapse 3-way conjunction high-ready∧low-coalesce∧low-rate). It tells you which splits to expect and why a shallow tree suffices (the partition has O(few) cells = O(few) regimes, not arbitrary depth). It also illuminates the sentinel columns formally: server_rows_per_forward is the in-band image of DOF-F / the §3.2 forward shape and rtt_us is the §6 latency-to-fixed-point signal — both sentinel-0 today, so a tree simply never splits on them now and 'lights up' once those channels (the very DOFs §3/§4 say the controller is moving) are observable. Net: C2 is the model-aligned class for catching the collapse basin's footprint, the genuine gap C1 leaves.

## C3 — Temporal sequence model (GRU / 1-D temporal CNN over the tick stream)

**Formal object.** Under the model the gate becomes a policy over a HIDDEN-STATE-FILTERED estimate of the §3 timing state: π(h_k), h_k = f(h_{k-1}, x_k), where the recurrent state h is an online estimator of the latent coordinates the observation does NOT carry — the per-slot δ_park phases (§3.1, free positive nondeterminism), the in-progress server forward's accumulating shape (§3.2/§3.3 DOF-F 'frames accumulate, first observed at the next drain'), and crucially the CURRENT BASIN of the §5 bistable system (modal-R1 vs the metastable collapse tail). State = belief over §3's nondeterministic timing + §5 regime; action = DENY-only; dynamics = §3.3 closed loop with the §6 fixed point AND its rare non-recovering excursion (README); objective = dps, now legitimately a function of a NON-Markov-in-observables quantity because the publish cadence is irregular (§3.3 poll/once-per-reply, restated in 01-atomic notes).

**Reduction / algorithm fit.** The reduction is to FILTERING / state-estimation of a partially-observed feedback system, then a policy on the filtered state. Formal target: the model's §3 timing is bounded nondeterminism that is NOT observable instantaneously (δ_park free, S free in its band, the in-flight forward's shape hidden until the next drain), and §3.3's 'no recv during a forward → self-batching feedback' makes the plant's relevant state EXPLICITLY lagged — so the observable x_k is a delayed, jittered projection of the controlling state. This is the textbook setting where a recurrent estimator (Kalman/HMM/GRU as a learned filter) recovers a near-sufficient statistic. The Karp-style transformation: cast the tick stream as observations of a hidden Markov / state-space model whose latent is (eligible-set phase, server-forward shape, §5 basin) and whose emission is the IssueFeatures row; the GRU/TCN is the amortized filter for that HMM, the HMM being (the file's own noted) lighter alternative. The reduction SUCCEEDS precisely where C1/C2 fail: the §5-R4 'sticks past the timeout' metastable tail is a TEMPORAL/trajectory property — basin membership persisting over a window — which only a state-carrying model can detect. The reduction's cost is the file's stated data-hunger, now formally explained: estimating a free-nondeterminism latent (δ_park, S have no code-fixed law, §3/open-Q2) needs many trajectories to pin the emission model.

**DOF moved (depth>1 live; approximate per caveat).** Same actuation DOFs as C1/C2 (DOF-B, DOF-depth/D live under caveat, DOF-F via timing), but C3 can move them ANTICIPATORILY along the §3.3 feedback: because its hidden state estimates the accumulating-but-unobserved forward shape (DOF-F) and the δ_park-driven eligible-set trend (DOF-park), it can deny BEFORE the collapse footprint is visible in a single tick — actuating DOF-B/DOF-F on the predicted trajectory rather than the current sample. This is the only family member that can act on DOF-F's lag (§3.3) rather than its already-realized effect. Flag DOF-depth/D as approximate (model pins depth==1; the recurrent reorder-relevant state is non-vacuous only under chunk_floor / DOF-R reactivation).

**What the model adds vs v2.** The model turns C3 from 'last-resort denoiser' into the FORMALLY-INDICATED class for the one failure the README says actually matters: the metastable convoy tail that 'fails to recover and sticks'. §3.3 (self-batching lag) + §5-R4 (collapse basin) + the README empirical 'rare deep mostly-self-recovering excursion, ~2-4% of windows, the stall is the non-recovering tail' together say the controlling quantity is a LAGGED, BISTABLE latent — exactly a hidden-state estimation problem, which is C3's home and is provably outside C1's (convex, memoryless) and C2's (memoryless partition) reach. It also CORRECTS the v2 ordering rationale: v2 defers C3 because 'denoising jitter' is speculative; the model says C3's real job is not denoising but BASIN-TRAJECTORY estimation of the §5 bistable system — a different and load-bearing justification. Finally it sharpens C3's binding risk: §3/open-Q2 leave δ_park and S(shape) as code-unfixed free nondeterminism, so the latent emission law must be LEARNED from data, which is the formal root of the data-hunger the file flags.

## C4 — Behavioral cloning of a hindsight/oracle gate (imitation, any backbone)

**Formal object.** C4 keeps any backbone's policy form but replaces the label with a hindsight target. Under the model the counterfactual label_counterfactual_gate_advantage IS the per-state ADVANTAGE function of the gate action under the §3.3/§5/§6 dynamics: A(s, deny) = E[dps | flip thread-t gate to deny at this state] − E[dps | all-allow], estimated by the deterministic-fixture paired runs (the model's §2 conforming closed system reproduces bit-identically, so the control arm is exact, not estimated — §2 + the 01/02 'default reproduces byte-for-byte'). The greedy hindsight oracle (b) is a 1-step surrogate: label deny where holding produced a fatter next §3.2 forward (coalesce_degree_inst jump = DOF-B up) without a leaf_rate loss (no §5-R3 starvation). So the formal object is: state = §3 projection, action = DENY-only, dynamics = §3.3 closed loop, objective = the EXACT terminal dps — and the label is the causal effect of the action ON the §5 basin transition / §6 fixed-point location.

**Reduction / algorithm fit.** The reduction is the deepest the model supplies, and it is to a CAUSAL-EFFECT-ON-AN-ABSORBING-CHAIN estimator. Cast the §5 regimes as states of a Markov chain over coalescing regimes (R1 modal ⇄ collapse-tail, with the collapse-tail being a slowly-/non-mixing near-absorbing set per the README 'sticks past the timeout'); the gate action biases the transition kernel of that chain (DENY raises DOF-B, pulling probability mass toward R1 and away from the collapse basin). label_counterfactual_gate_advantage is then exactly the do-calculus contrast on this chain's absorbing-state / long-run-dps functional — and the deterministic fixture makes do(gate=deny) a literal interventional sample, not an observational one. So C4 = behavioral cloning of the optimal policy of a Markov DECISION process whose dynamics are §3.3/§6 and whose hardest-to-see state is the §5-R4 metastable basin; the imitation step is ordinary supervised classification ONCE the interventional label exists. This formally validates C4 over C1-C3's observational labels: the model's §6 negative feedback CONFOUNDS observational credit (the system self-corrects regardless of the gate, so 'allow looked fine' is mostly the fixed point recovering on its own, README '~96-98% of excursions self-recover') — the convex/partition learners then fit the self-recovery, not the gate's effect. do(gate) breaks that confound; it is the model-mandated label.

**DOF moved (depth>1 live; approximate per caveat).** C4 actuates the SAME DOFs as whatever backbone it clones (DOF-B; DOF-depth/D live under caveat; DOF-F via timing) — the novelty is the label, not the action. What the model adds on dof_moved: the counterfactual perturbation should be placed at the DOF-F lag boundary (§3.3) — perturb the gate at a tick whose effect lands on the NEXT forward's shape — so the paired-run advantage isolates the DOF-B→DOF-F→§6-fixed-point chain rather than within-tick noise. Approximate-caveat flag: the model's depth==1 makes a single-tick counterfactual's effect propagate only through DOF-B; under chunk_floor (depth>1, DOF-depth/D live), one flip can also alter the outstanding-message ordering (DOF-R reactivated), so the counterfactual covers a strictly larger action-effect surface than the model's §0 spine admits — collect labels in BOTH the drain-all and chunked regimes (the 03-supervised notes' coverage grid) or the advantage is mis-specified.

**What the model adds vs v2.** The model promotes C4 from 'expensive labeling bolt-on' to the FORMALLY-CORRECT label for this plant, by identifying the exact confound the observational labels suffer: §6's negative-feedback self-correction means the system reaches a good fixed point with OR without the gate most of the time (README: excursions mostly self-recover), so any observational target (label_future_leaf_gain, realized_dps_window, deny_effectiveness) is dominated by the plant's autonomous recovery, not the action's effect — a textbook confounded-credit setting the model makes precise. The deterministic fixture (§2 conforming closed system, bit-identical) is then exactly the do-operator that de-confounds. It also reframes 'the label is the family's binding constraint' (the 03-supervised headline) as a model claim: the binding constraint is that the controlled quantity is a near-absorbing metastable basin (§5-R4 tail) whose effect is only visible interventionally — so the family's ceiling is removed precisely by C4's interventional label, and the family's natural hand-off (03-supervised notes 'hand to the RL family') is, under the model, the hand-off from imitating the MDP's optimal policy to solving it online.

## NEW: C5 — Supervised basin / collapse-onset classifier (binary metastability detector head)  🆕 (new — surfaced by the formal lens)

**Formal object.** A dedicated supervised classifier whose TARGET is not 'was allowing good' but 'is this state within (or about to enter) the §5-R4-like coalescing-collapse basin' — i.e. a learned indicator 1{s ∈ B_collapse} or 1{the trajectory from s sticks in the collapse tail for ≥ H ticks}. State = the §3 projection (with ready_backlog_norm, coalesce_degree_inst, leaf_rate, cross_thread_spread as the natural coordinates of the §5 partition); action derived = deny the implicated thread(s) iff the collapse head fires, else all-allow (so off the basin it is byte-identical baseline, on the basin it actuates DOF-B to re-fatten). Dynamics = §3.3/§6 with the bistable §5-R4 tail; objective = dps, decomposed by the model into (uncontrolled R1 fixed-point dps) MINUS (collapse-tail dps loss) — C5 targets only the second term, the part §6's negative feedback does NOT self-fix (README metastable tail).

**Reduction / algorithm fit.** Reduction to RARE-EVENT / ANOMALY classification on the model's regime partition, transformed by the §5 + README structure: the collapse basin is a MEASURE-SMALL, well-characterized region (the README convoy fingerprint B:88→1, server rows/forward ~130→1-3, every forward into the warmed 64-bucket, forward-rate ~330→1200/s — a precise multi-feature signature) occurring in ~2-4% of pipelined windows. That is exactly the imbalanced-binary-classification setting where the right tools are anomaly/rare-event detectors (one-class boundary, focal-loss or class-weighted GBDT, isotonic-calibrated probability) rather than a balanced gate classifier. The Karp-style move: relabel the C1/C2/C4 dataset by the §5-R4 membership oracle (cheap offline: detect the README fingerprint in the recorded trajectory — high cross-message-coalescing collapse with sustained low rows/forward) and fit a calibrated rare-event head. This SEPARATES the two jobs the model says C1/C2 conflate: (job A) pick the R1 operating point — a balanced, easy, low-value task §6 mostly self-solves; (job B) catch the rare collapse — an imbalanced, hard, high-value task that IS the dps tail. The model's bistability is the formal reason to split them; a single C1/C2 head trained on a balanced 'was-allow-good' label will under-weight the 2-4% tail that carries the actual loss.

**DOF moved (depth>1 live; approximate per caveat).** When it fires, C5 moves DOF-B hard (force-deny the convoy-implicated threads to re-bank ready and re-fatten the §3.2 forward, climbing back out of the warmed-64-bucket the README shows the collapse falling into) and thereby DOF-F (restores the self-batching feedback the collapse defeated, §3.3). Off the basin it moves nothing (identity), which keeps the §6 fixed point and §2 equivalence intact where they hold. DOF-depth/D: approximate/caveat — under chunk_floor a force-deny stands the thread at inflight>0, reactivating the §0-dead D; the collapse may itself be a depth>1 / DOF-R-reactivated phenomenon the model's depth==1 spine cannot represent, so C5's TARGET lives partly in the superseded regime — flag strongly.

**What the model adds vs v2.** This candidate exists ONLY because the model + README expose a structure the unscaffolded v2 pass could not: the dps loss is concentrated in a rare metastable basin (§5-R4 tail), NOT spread over the operating range. Every v2 supervised candidate optimizes a balanced 'was-the-gate-good' label and therefore spends capacity on the part §6 self-corrects, under-serving the 2-4% collapse tail that the README names as the real stall. C5 is the model-indicated refactor: a calibrated rare-event head whose label is §5-R4 basin membership (or H-tick stickiness), composed WITH all-allow as the off-basin default so it is a strict superset of the byte-identical baseline (ADR-0009-honest). It also reuses the existing collection harness verbatim (relabel, don't recollect) and degrades to identity when uncertain, making it the lowest-risk way to attack the specific failure the formal model elevates.

## NEW: C6 — Pairwise/listwise LEARNING-TO-RANK gate over threads (which thread to deny), not pointwise classify  🆕 (new — surfaced by the formal lens)

**Formal object.** Recast the per-tick gate as a RANKING over the T threads of 'expected dps gain from denying THIS thread now', then deny the top-k by a budget. State = the joint §3 projection across all T DEALERs (the cross_thread_spread / cross_thread_skew coordinates are first-class here, not broadcast scalars); action = a deny-SET (subset of threads), not T independent bits; dynamics = §3.3 with the CROSS-THREAD coalescing of §5-R1 ('the single-threaded server coalesces, across threads, the ≤T messages that arrived during the prior forward'); objective = dps at the §6 fixed point, which §5-R1 says is set by HOW the ≤T cross-thread messages align into each forward — a fundamentally JOINT, not per-thread-separable, quantity.

**Reduction / algorithm fit.** Reduction to LEARNING-TO-RANK / top-k subset selection, transformed by the model's cross-thread-only batching fact (§5-R1, fidelity-req #3 'cross-thread-only batching'). The key formal observation the pointwise C1/C2/C4 framing MISSES: §3.3 + §5-R1 make throughput a function of the cross-thread ALIGNMENT of emissions into a shared forward (which subset of the ≤T threads coalesce together), so the gate's effect is JOINT across threads — denying thread A changes the value of denying thread B (they compete/cooperate for the same forward window). That is exactly the structure where pointwise classification is mis-specified and learning-to-RANK (pairwise RankNet / listwise LambdaMART, or a permutation-invariant set-transformer scoring head over the T rows) is the correct hypothesis class. The Karp-style transformation: from the paired-counterfactual data (C4), derive PAIRWISE preferences 'denying A beat denying B from this joint state' (cheaper than absolute advantages — only the sign/order is needed) and fit a ranker; the deny-set is the top-k under a coalescing budget chosen so the surviving allowed threads still reach the §5-R1/§6 fixed-point batch without tipping into §5-R3 starvation. Permutation-invariance is mandated by the §2.2/§2.3 DEALER symmetry (same as C1's parameter-sharing, now applied at the SET level). This strictly generalizes C1/C2: a pointwise gate is the special case of a ranker with a per-thread-independent score and a fixed threshold instead of a budget.

**DOF moved (depth>1 live; approximate per caveat).** C6 moves DOF-B and DOF-F JOINTLY and DELIBERATELY: by choosing WHICH subset to deny it shapes the cross-thread coalescing of the next forward (§5-R1), the one place §5 says throughput is actually determined — it is the only family member that treats DOF-F's cross-thread alignment as the controlled variable rather than a per-thread side effect. It moves DOF-depth/D per-thread as the others do (approximate, live under chunk_floor). Flag: §5-R1's cross-thread coalescing is model-accurate (it has NOT moved since commissioning — caveat says the protocol/§3 timing/regimes are still accurate), so C6's joint-objective grounding is on the SOLID part of the model; only the per-thread depth>1 reading is the superseded/approximate part.

**What the model adds vs v2.** C6 is suggested directly by a model fact the v2 pass structurally could not use: the v2 supervised family is built ROW-WISE (one (tick,thread) sample, pool context broadcast in) — a per-thread-SEPARABLE assumption. But §5-R1 + §3.3 prove the objective is JOINT across threads (cross-thread-only coalescing into a shared forward), so the per-thread-independent label is mis-specified at the margin: the true target is which deny-SET best aligns the §5-R1 cross-thread batch. Learning-to-rank / set-scoring is the model-indicated class for a joint subset objective, and it cleanly inherits the §2 DEALER permutation symmetry as set-invariance. This both EXPLAINS a latent weakness in C1/C2 (independent per-row scoring cannot represent the deny-A-changes-value-of-deny-B coupling) and BROADENS the space toward the ranking/combinatorial-selection literature the v2 analogy-driven pass never reached — while resting its objective on the part of the model the caveat leaves intact.
