# C++ Component Soundness Audit — Prioritized Report

Scope: "spends a resource for no gain" defects across the node arena, search core, featurization, and producer transport. All severities are calibrated to wasted work, not data corruption; no correctness/equivalence hazard was found by any auditor.

## Dedup notes

- **Findings 2 and 9** were each reported by two auditors. The search-core auditor and the transport auditor both flagged the per-leaf `eval_finish` double-vectors (`gumbel.cpp:324,329`); merged into **#2**, with the transport auditor's lower-severity "adjacency" framing folded in. Both auditors agree the *bytes are consumed by the prior build* — so #9 (the wire-decode logits vector) is **churn-reduction, not spend-for-no-gain**, and is ranked accordingly.
- **Finding 8** (mis-named/over-margined admission estimate) and **#1** (O(K) resident) are linked: the estimate bakes the O(K)-per-tree assumption into the guard, so fixing #1 changes the correct calibration for #8.

## Highest-leverage items (why + fix)

1. **#1 — O(K) live arenas (HIGH, inspection-clear).** Resident RSS scales with the server batch width K, but the actual in-flight work is `inflight_msgs*msg_rows`; every parked slot holds a faulted-in 32KiB inline floor + overflow + a live Belief that buys no concurrency. This is the maintainer's exact soundness criterion (RSS spent for no added throughput). Fix: a fixed-size `ArenaPool` sized from the in-flight budget that slots check out/in, making O(K) resident unrepresentable. Inspection-clear; no measurement needed to act, though a before/after RSS reading would quantify the win.

2. **#2 — `eval_finish` re-pays the removed malloc bucket (MEDIUM, origin measured).** The serial path deliberately routed `logits_d`/`prior_scratch` through `ws_` after a K=64 wire profile flagged them as ~20% of malloc; the cursor — now the production engine — re-allocates them fresh per leaf and ignores the workspace it already owns. The *bucket's* significance was measured; the fix (thread `FeatureWorkspace&` into `eval_finish`, mirroring `eval_build_features`) is inspection-clear and structural — it makes a per-leaf-allocating evaluate-half unrepresentable.

3. **#4 — K-fold geometry replication (MEDIUM, inspection-clear).** `GeometryFeatures` is a pure function of env geometry with zero per-tree input, yet each of K builders recomputes and stores the same ≤67×67 table (~40 MiB at K=1024). Single-home it as an immutable precomputed table on the shared Environment; because it's immutable-shared it also removes the per-builder mutable memo's single-thread-ownership constraint. Inspection-clear.

4. **#5 — per-leaf `collected_features()` malloc (MEDIUM, DHAT-consistent).** The one per-leaf temporary that slips past the `FeatureWorkspace` amortization the surrounding code was built to provide — a fresh `vector<double>(20)` malloc/free on the hottest path, consistent with the per-leaf malloc DHAT observed. Fix: write the indicator straight into `out[]` or add a workspace slot. Inspection-clear; DHAT already corroborates the allocation exists.

5. **#6 — redundant `outstanding_` set on the real-producer path (MEDIUM, inspection-clear).** A per-message set node malloc/free backing `any_outstanding()` (never called by the real producer) and a desync check the drivers already perform via their authoritative `corr_to_group`. Second home for a fact the caller owns, zero added safety. Fix: collapse to a scalar "expected corr" on the coupled path / defer to the driver's map on the multiplexed path; confine `any_outstanding()` to the synthetic producer.

**Measure before acting (ADR-0009):**
- **#9** — the per-leaf logits-vector *count* is measured, but the *win* from pooling/flattening is conjecture; gate on a DHAT/bench delta. Note the bytes are used, so this is churn, not waste.
- **#8** — needs a fresh per-tree resident re-measurement for the cursor before rebuilding the fit (don't reuse fiber numbers).
- **#1** — actionable on inspection, but a before/after RSS reading would quantify and confirm the O(in-flight) bound.

**Honest conjecture flags:** #9's win is conjecture; #11/#12/#14 are structural/cleanliness items with negligible runtime cost (noted for completeness, not as resource wins); #14 is explicitly dominated by #1 and warrants no isolated action. No findings were invented beyond the four auditors' reports.
